## 2024-03-20 - Prevent Socket.io Array Broadcast Abuse and DoS
**Vulnerability:** The socket.io event handlers for joining rooms and WebRTC signaling (`join-room`, `offer`, `answer`, `ice-candidate`) accepted IDs directly from client events without validation. Passing arrays or extremely long strings to `socket.to(toId)` or `socket.join(roomId)` could cause unexpected broadcasting behavior (array abuse) or potential Denial of Service (DoS) due to memory consumption or processing overhead with enormous strings.
**Learning:** Socket.io's `to()` and `join()` methods can accept arrays or non-string values. If user input isn't strictly validated as a string with a reasonable length limit, attackers can manipulate these inputs to broadcast messages inappropriately or cause server strain.
**Prevention:** Always validate all arguments received from socket.io client events. Ensure IDs are strictly of type `string` and bounded to a safe maximum length (e.g., `<= 100` characters) before using them in socket.io operations.
