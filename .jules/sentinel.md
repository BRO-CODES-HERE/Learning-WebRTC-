## 2024-05-28 - Input Validation for Socket.io Event Arguments
**Vulnerability:** Socket.io's `socket.to()` can be vulnerable to Denial of Service (DoS) attacks if passed an array of IDs instead of a single string ID. This is because Socket.io attempts to iterate over all elements and send the event, which an attacker can exploit by sending a massive array to consume server resources and block the event loop. The vulnerability existed because input to the custom socket events (`join-room`, `offer`, `answer`, `ice-candidate`) in `server.js` was completely unvalidated before being used in `socket.to()` and `socket.join()`.
**Learning:** Even internal signaling events via websockets need their inputs rigorously type-checked and length-bounded. You can't assume that the client sending `roomId` or `toId` is the legitimate client application; it could be a malicious script sending unexpected types like arrays or massive strings.
**Prevention:** Always validate that incoming parameters intended to be single identifiers are explicitly strings (`typeof id === 'string'`) and fall within expected length bounds (e.g., `<= 100`) before passing them to core library functions like `socket.to()` or `socket.join()`.
